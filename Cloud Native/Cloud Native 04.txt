■ Reconciler 패턴
    - 클라우드 네이티브 인프라를 관리하는데 사용되거나 확장할 수 있는 패턴
    - CNI의 두 가지 상태 (현재 상태, 원하는 상태)를 강제할 수 있다. 
    - 4가지 규칙 
        + 1.모든 입출력에 대해 데이터 구조체를 사용한다
            (단, 데이터 구조체와 코드 베이스가 강하게 결합하지 않아야 한다)
        + 2. 데이터 구조체가 불변임을 보장한다 --> 안정적인 작업 가능
            ( 인프라 애플레케이션의 핵심 요소는 표현을 자원과 매핑하는 것)
            ( 데이터 구조체 업데이트기 필요하면 새로운 데이터 구조체를 생성해야 한다) 
        + 3. 자원 맵을 단순하게 유지한다.
        + 4. 실제 상태를 기대되는 상태와 일치시킨다.

■ Reconciler 메소드
    - GetActual() : 인프라스트럭처의 실제 상태 질의 , 감사
    - GetExpected() :  의도한 세계의 상태를 데이터스토어에서 읽는다.
    - Reconcile() : 세계의 기대되는 상태와 현재 상태를 반환, 상태의 조정
    - Destory() : 인프라를 삭제, 자원을 남겨두고 비활성화시키는 것이 아님

    - GetExpected() -->  GetActual --> Reconcile
